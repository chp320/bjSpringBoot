24.1 트랜잭션이 중첩된 상황의 이해
- [티켓 구입 서비스]에 두 개의 DAO가 있고 이를 트랜잭션 B로 묶음 (테이블: transaction1, transaction2)
- [로그 작성 서비스]에 한 개의 DAO가 있고 별도 트랜잭션 처리 없음 (테이블: transaction3)
- [티켓 구입 & 로그 작성 서비스]는 '티켓 구입'과 '로그 작성' 두 개 서비스를 각각 부르고 트랜잭션 A로 묶음
-> 결국 트랜잭션 A에 트랜잭션 B가 포함되는 상황
-> 티켓 구입 서비스. 단위 테스트는 정상이나 로그 작성 서비스. 에서 오류가 있을 경우, 전체 테스트에서 '로그 작성'으로 롤백되는데 트랜잭션 B 오류로 잘못 접근 가능성 존재
